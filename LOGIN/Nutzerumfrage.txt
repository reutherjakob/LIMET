==== Sidequest: NUTZERUMFRAGE ====
=== IDEE ===
Die Nutzenden eines Projektes werden uns diverse Angaben machen.
Um diesen Prozess nicht via Excel-Datein in E-Mail-Anhängen abhandeln zu müssen (in verschiedenen Versionierungen ausgefülkt von x Unbekannten mit erheblichem administrativem Aufwand),
soll eine Webseite entstehen.

=== Anforderungen an Website ===

1) Login System und Rollen

-> Nutzer sollen sich einloggen können. Dafür wird ein Login System erschaffen.
Siehe Tabelle users.sql


Limet-RB Subseite soll den Labor NutzerInenn so zugänglich gemacht werden, restliches RB nicht.
Dies wird über die zugewiesenen Rollen je Account umgesetzt.
Je nach Rolle sind bestimmte Seiten aufrufbar und Aktionen ausführbar.
[# 0: admin, 1: internal_rb_user, 2: external_rb_user, 3: spargefeld_ext_users]

- Verschiedene NutzerInenngruppen je Bereich bekommen dann jeweils einen Account angelegt. Account Name = Bereichsname.
Die table_users wird also auf Basis der Nutzergruppen befüllt, die Accounts bekommen alle Rolle 3 und können nur die Umfrage sehen und bearbeiten.

Im Zuge dessen soll auch das interne Login System aktualisiert werden.
Der derzeitige Vorschlag, die Nutzer Logins zu implementieren, könnte aber bis zur internen Umstellung bereits genutzt werden.



2) Informations Einholung
- Raumbuch basierend auf RUF anlegen.
- Festelegen, welche Nutzergruppe zu welchen Raumbereichen korrespondiert.

- Die Nutzenden eines Bereichs sollen dann für jeden Raum Informationen angeben können. Welche genau -> Aleks
- Diese werden in der Tabell Nutzeranforderugnen erfasst.
- Die Bearbeitungszeit ist zeitlich begrenzt.
- Es soll default Werte im Formular geben. Das Überschreiben dieser erfragt eien Begründung  -> Mit Aleks erarbeiten
- Es soll ein möglicher Raumbezug abgefragt werden, sprich welcher Raumsiedelt ggf. wohin.
